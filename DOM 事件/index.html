DOM

一，DOM 事件类

1,事件级别

DOM0 ： element.onclick = function(){};

DOM1: 没有涉及事件相关

DOM2: 1 element.addEventListener('click',function(),'false (false: 冒泡(默认) 或捕获：true)') ；IE： element.attchEvent('click',function(),'false (false: 冒泡(默认) 或捕获：true)')
          2 多次绑定后执行的顺序是不一样的，attachEvent是后绑定先执行，addEventListener是先绑定先执行

DOM3: element.addEventListener(' keyUp',function(),'false')

2,事件模型

事件捕获：从上到下；

事件冒泡：从下到上；

阻止冒泡和捕获： e.stopPropagation()，IE则是使用e.cancelBubble = true

取消默认事件： e.preventDefault()，IE则是使用e.returnValue = false; 什么元素有默认行为呢？如链接，提交按钮 input type=”submit” 等。当Event 对象的 cancelable为false时，表示没有默认行为，这时即使有默认行为，调用preventDefault也是不会起作用的。

3，事件流---- 浏览器与页面的交互

一阶段：捕获阶段

二阶段：目标阶段

三阶段：冒泡阶段

流程： 事件通过捕获到达目标阶段，在从目标阶段传入到冒泡阶段

事件捕获的流程： window --- --document----- html ---- body ---- 父级元素--子级元素----.....--- 目标对象

事件冒泡的流程： 和捕获相反
