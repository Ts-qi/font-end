盒模型

一，分类？

盒模型 有两种：
1 标准盒模型；
2 IE 盒模型；

二，组成？

组成： content border padding margin;

三，区别？

从两图可以看出： 标准盒模型的 width 和 height 是指content 的宽度和高度,而 IE 的 width 和 height 包括 padding ,border ,content

四，js如何获取width 和 height ？

标准盒模型的width 和height :

dom.style.height/width = xxxx;

IE盒模型的width 和height :

dom.currentStyle.width/height = xxx;一般不用 ，在火狐的兼容性不好；

window.getComputedStyle(dom).width/height = xxx; 通用性比较的好；一般用这个 ;

dom.getBoundingClientRect().width/height; 用于获取某个元素相对于视窗的位置集合。集合中有top, right, bottom, left,x ，y ,height,width 等属性；

五，盒模型 涉及到 边距重叠的问题？

BFC:边距重叠解决方案（块级格式化上下文）；

1 ,出现情况：

1 父子元素边距重叠：当父元素为非BFC时，第一个子元素的margin-top会重叠到父元素的margin-top

2 兄弟元素边距重叠：当两个相邻的元素为非BFC时，前者的margin-bottom和后者的margin-top会重叠；

3 空元素边距重叠：当空元素设置了margin-top和margin-bottom时，两个margin会重叠；

2, BFC 概念：

块级格式化上下文，它是指一个独立的块级渲染区域，只有Block-level BOX参与，该区域拥有一套渲染规则来约束块级盒子的布局，且与区域外部无关

3, BFC 原理：

1 BFC区域的元素不会影响外面的元素，反之亦然。

2 BFC区域不会与float元素发生重叠

3 计算BFC高度时，浮动元素能够撑开BFC的高度，达到清除浮动的效果。

4, BFC 创建：

1 overflow 除了 visible 以外的值 (hidden、auto、scroll)

2 float 值不能为 none

3 设置position不为relative和static（absolute，fixed，sticky）

4 设置display为table-cell，flex，inline-block

5, BFC 的应用：

可以解决边距重叠问题，当不想让两个元素相互影响时，可以对两个元素创建BFC，这样就不会发生边距重叠的问题。对于父子元素边距重叠问题，只需对父元素创建BFC，则可解决父子重叠
